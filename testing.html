Testing and measurement

How the campaign tests one subject-line decision, evaluates it by first-task completion, and separates message performance from incremental campaign impact.

Test design Defines the hypothesis, the two subject-line variants, and what stays fixed.

Hypothesis: a task-led subject line will drive a higher seven-day completion rate than a capability-led one. Only the subject line changes; audience, body, preview text, CTA, send time, and destination stay fixed.

VariantATask-led

Subject: Try Junie on one clear task

Frames the first step as a single bounded, manageable action.

VariantBCapability-led

Subject: See Junie handle a real project task

Leads with what Junie can do: plan, change, test, and report on a real task.

How the result is evaluated Shows which outcome selects the winner and which signals help explain it.

  • Primary metric: first Junie task completed within seven days.

    This determines the winning variant.

  • Diagnostic path: qualified CTA click → Junie opened → task submitted → task completed.

    This shows where the two variants begin to diverge.

  • Guardrails: unsubscribes · spam complaints · delivery failures.

    Open rate adds context but does not determine the winner.

Decision rule Sets the condition for rollout or another test.

Reliable, meaningful lift in seven-day completion with stable guardrails

roll out the stronger variant as the new control.

Inconclusive result

keep the current control and test a sharper messaging contrast.

Incremental impact Separates message performance from the effect of running the campaign at all.

The A/B test identifies the stronger subject line among users who receive the campaign.

A randomly assigned no-send holdout estimates whether the campaign lifts first-task completion beyond what would have happened through normal product use.

Sample size, test duration, decision thresholds, eligibility rules, and event definitions would be set from the eligible audience and the current activation baseline.
