# Email Campaign Portfolio

A responsive static portfolio case study presenting an independent activation campaign concept for JetBrains Junie.

The project covers campaign strategy, lifecycle automation, testing and measurement, and a fully coded featured-email prototype. It is an independent portfolio project and is not commissioned by or affiliated with JetBrains.

## Pages

- `index.html` — campaign overview and featured email
- `strategy.html` — campaign and email strategy
- `lifecycle.html` — lifecycle automation
- `testing.html` — testing and measurement plan
- `email-01.html` — standalone coded email prototype

## Features

- Responsive multi-page layout
- Shared navigation and contact menu
- Light and dark email-preview modes
- Desktop and mobile email-preview controls
- Expanded email view
- GitHub Pages-safe relative paths
- No framework, build step, or server-side routing

## Project files

- `styles.css` — shared site styles
- `script.js` — navigation, preview controls, dialog, and interactive diagrams
- `email-01.html` — self-contained HTML email and preview-specific theme handling

## View locally

Open `index.html` in a browser. The site is entirely static and requires no installation or build command.

## Deploy to GitHub Pages

1. Add the project files to the root of a GitHub repository.
2. Commit and push the files to the repository's default branch.
3. In the repository settings, open **Pages**.
4. Choose **Deploy from a branch**.
5. Select the default branch and the repository root (`/`), then save.

Because all site links and assets use relative paths, the site works when GitHub Pages hosts it inside a repository subdirectory.
